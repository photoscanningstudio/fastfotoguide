How to Label and Name Your Scanned Photo Files

A good naming system turns a pile of anonymous files into a searchable family archive. Here's the convention that professional archivists use — adapted for families.

You've scanned 800 family photos over a long weekend. They sit in a folder on your desktop. The files are named scan_001.jpg through scan_800.jpg. You feel accomplished — and you should. But three months from now, when your cousin asks for the photo of Grandma and Grandpa at the lake house in 1978, you'll face a dispiriting reality: finding that specific image among 800 generically named files means scrolling through hundreds of thumbnails, squinting at each one, hoping you recognize it.

A thoughtful file naming convention prevents this. It turns your archive from a haystack into an index — searchable, browsable, and intuitive for anyone in the family, including people who inherit the archive decades from now.

The Foundation: Year-First Naming

The single most important element in a photo file name is the date, and it should always come first. Use the format YYYY-MM — four-digit year, two-digit month. This ensures that when files are sorted alphabetically, they automatically sort chronologically. A file browser shows 1972-06-Beach-Vacation before 1985-12-Christmas, which is exactly the order you'd naturally want to browse.

If you know the exact date, use YYYY-MM-DD. If you only know the year, use YYYY-00-00 or just YYYY. If you don't know the date at all, use a descriptive estimate like "1970s" as a prefix. Approximate dates are better than no dates — they at least get the photo into the right neighborhood.

The Complete Naming Pattern

The full naming convention follows this structure: YYYY-MM-Description-NNN.jpg. The year and month establish chronological order. The description identifies the event, location, or subject. The three-digit number (NNN) distinguishes individual photos within a group. Here are some examples of this pattern in action: 1978-07-Lake-House-Vacation-001.jpg through 1978-07-Lake-House-Vacation-024.jpg. Or 1985-12-Christmas-at-Grandmas-001.jpg. Or 1960s-Dad-Army-Portrait-001.jpg.

This naming pattern is self-documenting. Anyone who encounters the file — your cousin, your grandchild, a future family historian — can understand what the photo contains and when it was taken just from reading the file name. No database required, no software dependency, no special tools. The information lives in the file name itself, which persists across every operating system, storage medium, and backup service.

Naming During Scanning

The most efficient time to name files is during the scanning process itself. Scanning software — including the Epson FastFoto software — lets you set a batch name before each scan run. When you load a group of photos from the same event into the scanner, name the batch according to your convention: "1985-12-Christmas" for example. The software appends sequential numbers automatically, producing "1985-12-Christmas-001.jpg" through however many photos are in the batch.

This means your files are correctly named the moment they're created. No renaming pass required afterward. The pre-scan sorting you did — grouping photos by decade and event — pays off directly here, because each physical batch corresponds to a logical naming group.

Folder Structure: The Other Half

File names work best when paired with a clear folder hierarchy. Your top-level folder — "Family Photo Archive" — contains decade folders: "1950s," "1960s," "1970s," and so on. Inside each decade folder, create event or category subfolders. The folder path itself adds context: Family Photo Archive / 1980s / 1985-12-Christmas / contains the Christmas files, while Family Photo Archive / 1980s / 1987-06-Disney-Vacation / contains the vacation files.

For photos you can't date precisely, create a "Undated" or "Unknown-Date" folder within the appropriate decade. Better to have a clearly labeled "Undated" category than to assign incorrect dates that mislead future users of the archive.

Characters to Avoid in File Names

Keep file names compatible across all operating systems and storage services. Avoid special characters that cause problems: slashes, backslashes, colons, asterisks, question marks, quotation marks, angle brackets, and pipe symbols. Use hyphens or underscores to separate words instead of spaces — while modern systems handle spaces in file names, some older tools, scripts, and cloud services can choke on them.

Stick to letters, numbers, hyphens, and underscores. This combination works everywhere — Windows, Mac, Linux, Google Drive, iCloud, Amazon S3, USB drives, and every other storage system your files might touch over the coming decades.

Adding Metadata Beyond the File Name

File names carry basic identification. For richer metadata — people's names, relationships, locations, stories behind the photo — use the tools available in your photo management software. Google Photos, Apple Photos, and DigiKam all let you add descriptions, tags, and captions to individual images. This metadata is searchable within the application and, in many cases, embedded in the file's EXIF data so it travels with the photo.

You don't need to add metadata to every photo. Focus on the most important images — the ones that future family members will want to understand. A caption on Grandpa's military portrait explaining which unit he served in and what year it was taken adds irreplaceable context that the file name alone can't convey.

Batch Renaming Existing Files

If you've already scanned hundreds of photos with default names like IMG_0001 through IMG_0800, don't panic. Free batch renaming tools can rename entire folders of files according to a pattern you define. On Windows, PowerRename (part of Microsoft PowerToys) handles batch renaming with preview. On Mac, the built-in Finder rename feature works for simple patterns. For more complex renaming, XnView MP and Advanced Renamer are both free and powerful.

The key is to rename batch by batch, not file by file. Move photos from the same event into a temporary folder, batch-rename them with the appropriate convention, then move them to their permanent location in your folder structure. Working in groups of 20 to 50 files makes the task manageable even for large archives.

The 30-Second Rule

Here's the test for any naming and organization system: if a family member who has never seen your archive opens the folder, can they find a specific photo within 30 seconds? If the answer is yes — because the folders are logically structured and the file names are descriptive — your system works. If the answer is no, simplify and clarify until it does. Your archive isn't just for you. It's for everyone in the family, today and for generations to come.
